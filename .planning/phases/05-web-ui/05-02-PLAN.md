- Export `useJobPoller(jobId: string | null, options: { onComplete: (data: JobResponse) => void, onError: (data: JobResponse) => void })`.
- Uses `useEffect` with `jobId` as dependency.
- Inside: `setInterval` at 1500ms calling `api.getJob(jobId)`.
- On `data.status === 'complete'`: `clearInterval`, call `onComplete(data)`.
- On `data.status === 'error'`: `clearInterval`, call `onError(data)`.
- Returns `{ status: string | null, data: JobResponse | null }` — updated on every poll tick.
- CRITICAL: `return () => clearInterval(id)` in the useEffect cleanup. Store interval ID in a `useRef` so cleanup always references the live interval.
- Use a `pollingKey` number prop (default 0) — when it changes, the hook restarts polling. This supports doc-type override re-extraction.
