## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding within cells, badge padding |
| sm | 8px | Input internal padding, button icon-to-label gap |
| md | 16px | Table cell padding, form field spacing, card padding |
| lg | 24px | Section separation (header table / line items table), container padding |
| xl | 32px | Upload zone internal padding, progress area vertical padding |
| 2xl | 48px | Main content max-width margin, vertical rhythm between major blocks |
| 3xl | 64px | Page-level vertical centering offset for upload and progress views |

Exceptions:
- Touch/click targets for editable cells: minimum 44px height (accessibility — WCAG 2.5.5)
- Upload drop zone: minimum 160px height to provide clear drag-and-drop affordance
- Spinner container: 80px centered block (fixed height prevents layout shift on status text change)

**Source:** 8-point grid default. Exceptions derived from WCAG requirements and CONTEXT.md upload zone expectations.

## Interaction Contract

All interaction decisions below are locked (source: CONTEXT.md). Executor must not deviate.

### Upload Zone

- Accepts: PDF, XLSX, XLS, PNG, JPG, HTML
- Implementation: `react-dropzone` `useDropzone` hook (not native HTML5)
- Drag-over state: `border-primary` outline + `bg-primary/5` fill transition
- File validation feedback: inline text below drop zone (not toast)
- No persistent upload zone after submission — flow is linear

### Progress Indicator

- Layout: vertically centered in the card, `Loader2` icon with `animate-spin`
- Status text stages (exact copy — see Copywriting Contract below)
- Source: poll `GET /jobs/{id}` `.status` field every 1500ms (1.5s — midpoint of 1-2s range)
- Polling: `setInterval` in `useJobPoller` hook with `clearInterval` cleanup on unmount and jobId change
- Stops when `status === 'complete'` or `status === 'error'`

### Review Table — Header Fields

- Two-column: Label column (left, width ~35%) | Value column (right, width ~65%)
- Label cell: `text-label font-mono` — not editable, no hover effect
- Value cell: `hover:bg-muted/50 cursor-pointer` — EditableCell component
- "Not found" value: `text-muted-foreground italic` — click-to-edit clears placeholder, shows empty input

### Review Table — Line Items

- Separate multi-column table below header table — not tabs, not collapsible
- Only rendered for doc types: `purchase_order`, `invoice`, `supplier_comparison`
- Column headers: schema-derived field names (human-readable via `fieldLabels.ts`)
- Each cell is an EditableCell

### Inline Edit (EditableCell)

- Click anywhere on value cell → switches to `<input autoFocus>`
- Save trigger: blur (click away) OR Enter key
- Cancel trigger: Escape key — reverts to original value, no PATCH
- PATCH fires only if `draft !== value` AND `!(draft === '' && value === 'Not found')`
- PATCH fires immediately per-cell on save (not batched)
- Input style: `w-full bg-transparent border-b border-primary outline-none px-1`

### Document Type Override

- Displayed via shadcn Badge (read-only) + shadcn Select (editable) in DocTypeBar
- Dropdown visible only when `doc_type` is non-null (after classification)
- Selecting a new type fires `PATCH /jobs/{id}/doc_type` → restarts polling (polling key changes)
- Dropdown placement: above the review table, below the progress-to-review transition

### Error Banner

- Replaces the spinner area (same card, same position — no layout shift)
- Contains: `AlertCircle` icon + human-readable message + "Try again" button
- "Try again" resets state machine to `{ tag: 'upload' }` — no page refresh
- No toast notifications

### Post-Download

- After CSV download: "Upload another document" button appears
- Button resets state machine to `{ tag: 'upload' }`
- No persistent upload zone always visible

## Flow State Machine

The App.tsx state machine has exactly 4 phases. No back-navigation is permitted between phases.

```
upload → (file submitted) → processing → (poll complete) → review → (CSV downloaded) → done
                                       ↓ (poll error)
                                     error banner (inline, same card)
                                       ↓ (Try again)
                                     upload
```

| Phase | Component Rendered | Entry Trigger | Exit Trigger |
|-------|--------------------|--------------|--------------|
| upload | UploadZone | Initial / reset | File submitted → POST /extract success |
| processing | ProgressView | POST /extract returns job_id | Poll returns `complete` or `error` |
| review | DocTypeBar + ReviewTable + LineItemsTable (conditional) | Poll returns `complete` | "Download CSV" clicked |
| done | ReviewTable (readonly) + "Upload another document" button | CSV download triggered | "Upload another document" clicked |

**Error sub-state:** Handled within `processing` phase — ProgressView renders error banner when `status === 'error'`. No separate error phase in state machine.
