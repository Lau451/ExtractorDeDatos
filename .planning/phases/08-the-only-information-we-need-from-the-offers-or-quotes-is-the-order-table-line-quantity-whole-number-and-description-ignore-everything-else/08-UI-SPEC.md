## Component Behavior Contract

### ReviewTable Guard (Phase 8 net-new interaction)

**Trigger:** `phase.tag === 'review'` AND `jobData.doc_type` is `'tender_rfq'` or `'quotation'`

**Behavior:** `<ReviewTable>` is NOT rendered. The "Extracted Fields" section is fully absent from the DOM — no placeholder, no collapsed state, no empty table.

**What remains visible for tender_rfq / quotation:**
1. `<DocTypeBar>` — doc type label + override dropdown (always shown, no change)
2. `<LineItemsTable>` — item_number, quantity, description columns (shown when `line_items` array has entries)
3. "No line items" empty state (rendered by `LineItemsTable` when `items.length === 0`)
4. Download CSV button
5. Export-warnings banner (if `X-Export-Warnings` header is present)

**Set of doc types that suppress ReviewTable:** `['tender_rfq', 'quotation']`
Name the constant: `LINE_ITEMS_ONLY_DOC_TYPES` (a `Set<string>` in `App.tsx` or `@/lib/docTypes.ts`).

### LineItemsTable — unchanged

The `LineItemsTable` component requires no changes. It already:
- Derives column headers from `Object.keys(items[0])` — will correctly show item_number, quantity, description
- Renders an empty state ("No line items") when `items.length === 0`
- Renders `EditableCell` per cell (inline editing of line items remains available)

### DocTypeBar — unchanged

The doc type display and override dropdown are shown for all doc types including tender_rfq and quotation. No changes.

### Download CSV button — unchanged

Button text: "Download CSV" (idle), "Downloading..." (in-flight with Loader2 spinner). No changes to copy or behavior.

## Interaction States

### Review phase — tender_rfq or quotation

| State | What the user sees |
|-------|--------------------|
| Line items extracted (≥1 item) | DocTypeBar + LineItemsTable with item rows + Download CSV button |
| Zero line items extracted | DocTypeBar + "No line items" text + Download CSV button |
| Download in flight | Download CSV button disabled, "Downloading..." label with Loader2 spinner |
| Export warnings present | Warning banner below Download CSV button; phase stays in review; user can re-download |
| Doc type overridden | Transitions to processing phase, re-polls, returns to review with new doc type result |

### Review phase — other doc types (purchase_order, invoice, supplier_comparison)

Unchanged from Phase 7. ReviewTable is shown for these types.
