<objective>
Fix the status collision that causes 409 errors when exporting completed jobs.

Purpose: `set_complete()` in ingestion/service.py sets status='complete' before the extraction pipeline runs, but the pipeline immediately overwrites status to 'classifying'. If the user polls or exports during this window, the job appears incomplete (409). The fix: ingestion stores raw_text without setting status to 'complete' — only `set_extraction_result()` at the end of the full pipeline should mark a job as complete.

Output: Patched ingestion flow where 'complete' status is only reached once extraction_result is populated.
</objective>
